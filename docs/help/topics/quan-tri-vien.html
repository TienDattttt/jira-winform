Quản trị viên

Áp dụng cho Project Manager, Admin dự án và System Admin.

Quản trị dự án

Các cấu hình ở mức dự án chỉ áp dụng cho project hiện hành.

Nhóm chức năng Mục đích
Members / Roles Thêm thành viên, đổi vai trò project, loại bỏ thành viên không còn tham gia.
Board Quản lý cột board, ánh xạ status, WIP limit và cách hiển thị công việc.
Workflow Quản lý status và transition cho issue trong dự án.
Permissions Kiểm soát quyền xem, tạo, sửa, xóa issue và các thao tác trọng yếu.
Labels / Components / Versions Chuẩn hóa metadata dùng cho phân loại, release và báo cáo.
Thay đổi workflow hoặc permission có thể ảnh hưởng trực tiếp tới board, issue details và báo cáo. Chỉ áp dụng sau khi đã xác định rõ tác động nghiệp vụ.

Quản lý sprint

  1. Tạo sprint để gom nhóm issue cần thực hiện trong một chu kỳ làm việc.
  2. Start sprint khi phạm vi đã chốt.
  3. Close sprint khi kết thúc; issue chưa hoàn thành có thể trả về backlog hoặc chuyển sang sprint khác.

Quản trị hệ thống

Chức năng này dành cho tài khoản có role hệ thống Admin.

  • Quản lý người dùng: tạo tài khoản, chỉnh thông tin, reset mật khẩu, khóa hoặc mở khóa.
  • Gán role hệ thống cho người dùng theo nhu cầu vận hành.
  • Theo dõi khả năng truy cập menu và project theo membership thực tế.
Quyền Admin hệ thống không thay thế membership của project. Muốn thao tác trong một project, người dùng vẫn cần là thành viên của project đó.

Tích hợp và vận hành

Thành phần Mục đích Lưu ý vận hành
GitHub / Confluence Kết nối issue với tài nguyên ngoài hệ thống. Kiểm tra thông tin xác thực và quyền truy cập trước khi bật cho người dùng.
Webhook Gửi sự kiện ra hệ thống khác khi issue hoặc project thay đổi. Luôn test endpoint và theo dõi delivery history sau khi cấu hình.
API token Cho phép truy cập API theo phạm vi được cấp. Token chỉ hiển thị giá trị lúc tạo; cần thu hồi khi không còn sử dụng.
Email notification Gửi email khi có sự kiện liên quan tới issue hoặc assignment. Phụ thuộc vào SMTP được cấu hình đúng.

Khuyến nghị vận hành

  • Không chỉnh permission hoặc workflow trực tiếp trên production mà không có đánh giá tác động.
  • Kiểm tra delivery history sau mỗi lần thay webhook hoặc secret.
  • Chuẩn hóa labels, components và versions để báo cáo không bị phân mảnh dữ liệu.
  • Định kỳ rà soát role hệ thống và membership dự án để tránh cấp quyền dư thừa.
