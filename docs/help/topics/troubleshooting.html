Xử lý sự cố

Các lỗi thường gặp khi vận hành client desktop và hướng kiểm tra ban đầu.

Không đăng nhập được hoặc không thấy project

  • Kiểm tra tài khoản còn active và mật khẩu đã được đặt đúng.
  • Kiểm tra user có membership trong project cần thao tác.
  • Nếu dùng OAuth, xác nhận các endpoint và thông số issuer hoặc JWKS đã được cấu hình đúng.

Lỗi kết nối cơ sở dữ liệu

  • Xác nhận biến JIRACLONE_ConnectionStrings__Default đã được cấp và không bị sai tên instance, database hoặc credential.
  • Kiểm tra SQL Server có thể truy cập từ máy client qua mạng nội bộ.
  • Kiểm tra log ứng dụng tại thư mục logs để xem lỗi khởi tạo DbContext hoặc migration.

Lỗi attachment hoặc export

  • Nếu upload thất bại, kiểm tra App:StoragePath, dung lượng file và quyền ghi vào thư mục đích.
  • Nếu export PNG hoặc Excel thất bại, kiểm tra quyền ghi của thư mục được chọn và xác nhận file không đang bị ứng dụng khác giữ.
  • Nếu người dùng khác không thấy attachment, kiểm tra xem storage path có đang trỏ vào thư mục local thay vì thư mục dùng chung hay không.

Lỗi webhook hoặc tích hợp

  • Mở lịch sử gửi webhook để xem HTTP status code, thời gian gửi gần nhất và endpoint đang lỗi.
  • Kiểm tra URL endpoint, secret và khả năng truy cập mạng từ máy đang chạy ứng dụng.
  • Với GitHub hoặc Confluence, xác nhận repository, owner, space key hoặc base URL đã lưu đúng.

Lỗi mở file trợ giúp CHM

  • Nếu file CHM bị Windows chặn sau khi tải từ Internet, mở Properties của file và bỏ trạng thái Unblock.
  • Đảm bảo file Help\JiraClone.chm đã được copy cùng với thư mục chạy của ứng dụng.
  • Nếu pane Contents hiển thị sai ký tự, đóng hoàn toàn cửa sổ HTML Help rồi mở lại sau khi thay file mới.
