Troubleshooting

Hướng dẫn xử lý các sự cố thường gặp trong môi trường vận hành.

Không đăng nhập được hoặc không thấy dữ liệu

  • Kiểm tra lại tài khoản, mật khẩu và trạng thái khóa của tài khoản.
  • Xác nhận ứng dụng đang kết nối đúng môi trường dữ liệu.
  • Nếu không thấy project mong đợi, kiểm tra membership của người dùng trong project đó.
  • Nếu màn hình chỉ hiển thị trạng thái rỗng, kiểm tra project active và bộ lọc search hiện tại.

Ứng dụng không khởi động do lỗi database

  • Kiểm tra chuỗi kết nối đã cấp cho ứng dụng.
  • Xác nhận SQL Server đang chạy và máy trạm truy cập được tới máy chủ dữ liệu.
  • Xác nhận tài khoản kết nối có quyền phù hợp với môi trường.
  • Đọc log khởi động trước khi thao tác tiếp.

Không upload được attachment

  • Kiểm tra quyền của người dùng trên issue.
  • Kiểm tra dung lượng file và giới hạn cấu hình.
  • Kiểm tra storage path hoặc network share cho attachment.

Không xuất được Excel hoặc PNG

  • Xác nhận màn hình báo cáo đã tải xong dữ liệu.
  • Kiểm tra thư mục đích có quyền ghi.
  • Nếu xuất Excel thất bại, kiểm tra file đích không đang bị khóa bởi Excel hoặc LibreOffice.
  • Kiểm tra lại project và sprint đang chọn nếu dữ liệu xuất ra không đúng.

Không nhận được notification, email hoặc webhook

  • Kiểm tra assignee, watcher hoặc điều kiện phát notification.
  • Với email, kiểm tra SMTP và thông tin xác thực.
  • Với webhook, kiểm tra endpoint, secret, response code và delivery history.

Không mở được file trợ giúp

  • Kiểm tra file JiraClone.chm có tồn tại trong thư mục Help của bản cài đặt.
  • Đóng hoàn toàn cửa sổ HTML Help cũ trước khi mở lại file trợ giúp mới.
  • Nếu file vừa được chép từ nguồn ngoài, bỏ chặn file trong Properties nếu Windows đánh dấu là tải từ Internet.
  • Nếu vẫn lỗi, cài đặt lại ứng dụng để khôi phục artifact trợ giúp đi kèm.
Khi sự cố liên quan tới database, permission, webhook hoặc SMTP, người dùng cuối không nên tự thay đổi cấu hình. Hãy chuyển thông tin lỗi cho quản trị viên hoặc đội vận hành.
