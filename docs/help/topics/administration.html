Quản trị

Nhóm thao tác quản trị ở cấp project và cấp hệ thống.

Các màn hình quản trị của ứng dụng được tách theo hai lớp. Một lớp dành cho project admin hoặc project manager để quản lý project, board, workflow, webhook và thành viên. Lớp còn lại dành cho system admin để quản lý user hệ thống.

Các nhóm cấu hình chính

Nhóm Topic chi tiết
Thành viên, vai trò và permission Thành viên và phân quyền
Workflow, board column và WIP limit Workflow và cấu hình board
Webhook, GitHub, Confluence Webhook và tích hợp
API token cá nhân API token
Quản lý tài khoản hệ thống Quản lý người dùng hệ thống
Quyền quản trị hệ thống không tự động thay thế membership của project. Một tài khoản vẫn phải là thành viên project thì mới thao tác được dữ liệu của project đó.
